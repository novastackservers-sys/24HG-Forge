Anti-Cheat Breakdown

When a game doesn't work on Linux, it's almost always the developer's choice, not a Linux limitation. Here's how each anti-cheat system works:

Supported

VAC (Valve)

Works perfectly on Linux. Always has. CS2, TF2, Dota 2 — all flawless.

Developer's Choice

EasyAntiCheat (EAC)

Works if the developer enables Linux support. Rust, Apex Legends, Elden Ring, Palworld all work. Fortnite and Dead by Daylight don't — because their devs chose not to enable it.

Developer's Choice

BattlEye

Same story. Works if the dev opts in. DayZ and ARK: Survival Evolved work. Destiny 2 and R6 Siege don't — their developers declined.

Not Supported

Vanguard (Riot Games)

Does not work. Requires Windows kernel-level access. No Linux path exists. Valorant is the main affected title.

Not Supported

Ricochet (Activision)

Does not work. Kernel-level anti-cheat with no Linux support. Blocks Call of Duty titles.

Not Supported

FACEIT Anti-Cheat

Does not work on Linux. The FACEIT client requires Windows. Regular CS2 matchmaking works fine — just not FACEIT.

The bottom line: EAC and BattlEye both have official Linux support. Whether a game works depends on the game's developer flipping the switch. If a game you play is blocked, the best thing you can do is ask the developer to enable Linux support.

How to Check Your Games

Before you install

  • Search your games on ProtonDB.com — community-tested compatibility reports for every Steam game
  • Check AreWeAntiCheatYet.com — tracks anti-cheat Linux support status
  • Not sure about a specific game? Ask in our Discord #support channel

After you install 24HG Forge

  • Run forge-compat scan in a terminal — scans your Steam library and reports compatibility for every installed game
  • The built-in Game Compatibility Checker (v1.6) cross-references ProtonDB ratings, anti-cheat databases, and your hardware to give you a per-game verdict

Most games work. The few that don't are blocked by their developers, not by Linux.
